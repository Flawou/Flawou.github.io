Pour créer nos clés de sécurités SSH : 
Sur client : 

ssh-keygen (NON CONNECTER SSH)

type C:\Users\Flaw\.ssh\id_rsa.pub (pour copier la clé) 

ssh -i $env:USERPROFILE\.ssh\id_rsa user@192.168.0.30  ← POUR WINDOWS

puis on se connecte en ssh sur le compte utilisateur, aller dans .ssh\authorized_keys 
nano ~/.ssh/authorized_keys  Puis coller la clé de notre utilisateur windows

si .ssh pas créer faire : mkdir -p ~/.ssh       puis  chmod 700 ~/.ssh 

une fois cette étape terminer nous pouvons nous connecter sans pw :

ssh -i C:\Users\Flaw\.ssh\id_rsa user@192.168.0.30

ssh-copy-id user@192.168.X.X (NON CONNECTER SSH)  ← POUR LINUX
nano /etc/ssh/sshd_config

changer le port d'écoute

PermitRootLogin no

décommentez Port 22

modifier le port ( exemple 2222)

depuis client pour s'y connecter : ssh user@192.168.217.10 -p 2222
