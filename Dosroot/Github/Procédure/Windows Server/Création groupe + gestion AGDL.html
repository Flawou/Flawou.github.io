Création groupe + gestion AGDLP

Account Global Domain Local Permissions.

Nous commencer par créer nos OU pour avoir une gestion plus simple de nos groupes.

Après avoir créer notre OU Groupe, nous allons créer un groupe pour chaque service

Nous allons créer un second groupe, cette fois-ci pour l'administration de droit de ces groupes

pour créer le premier groupe aller dans l'OU groupes puis cliquer ici :

Choisir le nom du groupe et son étendu, pour ce premier groupe, global.

étant donner que son groupe est Globale, nous allons le nommer GG_ suivis du nom du groupe.

Une fois nos GG créer, nous allons créer nos groupes de Domaine local.

Cette fois-ci comme on sélectionne Domaine local, nous allons le nommer GDL_NomG_RW ou _RO

RW pour les droits d'écriture/lecture, RO pour les droits de lecture.

Une fois cette action terminée, nous allons imbriquer nos utilistateur dans nos GG. Pour celà, sélectionnez tous les utilistateur de l'OU concerné puis ajouter à un groupe. 

ici, GG_Dev étant donner que nos utilisateur ce trouve dans l'OU Développeur.

Ensuite, nous allons intégrer nos GG dans les GDL, pour ceci, clique droit sur le groupe voulu puis ajouter à un groupe. par exemple, ici, GG_Compta dans GDL_Compta_RW puis refaire l'action pour GDL_Compta_RO.

Ensuite, nous allons appliqué les droits NTFS des groupes concernés sur leur dossier dédié (en partant du principe que le mappage a déjà été fait sur le dossier de l'entreprise), pour celà : clique droit sur le dossier voulu (ici Dev), propriétés, puis sécurité.

Aller dans Avancé et désactiver l'héritage

Convertir les autorisations (Pour garder les perm d'application dessus)

Ensuite, Appliquer puis OK. 

Aller maintenant dans Modifier

 

Commencer par supprimer le groupe “Tout le monde” puis cliquer sur ajouter, mettre le groupe GDL du département concerné, ici nous sommes sur le dossier dev, donc GDL_Dev_RW pour qu'ils aient le droit d'écriture et lecture.

coché les autorisations en lien avec la GDL

Les utilisateurs de l'OU Dev (donc imbriqué dans GG_Dev, qui est lui même imbriqué dans GDL_Dev_RW) peuvent désormais avoir accès à ce dossier, créer leur propre fichier, y écrire, les lires et les exécuter.

 

 

Note : Les permissions NTFS offre un plus grand choix qu'un partage simple, de plus les permissions de celui-ci s'appliquent uniquement lorsque l'on accède à distance à ce dossier/fichier, alors qu'avec les permissions NTFS s'appliquent aussi lorsque on accède en local.

Bonus : En cas de cumul de droit ( NTFS + Partage ) les permissions effectives correspondront aux permissions les plus restrictives.
