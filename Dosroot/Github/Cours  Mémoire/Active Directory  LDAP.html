Active Directory / LDAP

 


LDAP (Lightweight Directory Access Protocol) et Active Directory sont deux concepts liés à la gestion des annuaires et de la sécurité dans les environnements informatiques Windows.

  1. LDAP (Lightweight Directory Access Protocol) : LDAP est un protocole de communication standard utilisé pour accéder et gérer des services d'annuaires informatiques. Il est principalement utilisé pour rechercher et récupérer des informations à partir de bases de données d'annuaires, telles que les annuaires d'utilisateurs, les annuaires d'entreprises et d'autres données de répertoire. LDAP est un protocole léger et efficace qui fonctionne sur le modèle client-serveur.
  2. Active Directory : Active Directory est un service d'annuaire développé par Microsoft. Il est utilisé pour gérer et organiser des ressources réseau, telles que des ordinateurs, des utilisateurs, des groupes, des imprimantes, des serveurs, et bien d'autres, au sein d'un environnement Windows. Active Directory est un composant clé des systèmes d'exploitation Windows Server et est utilisé pour l'authentification, l'autorisation, la gestion des politiques de sécurité et bien d'autres tâches liées à la gestion de réseau.

En relation avec Active Directory, LDAP est utilisé comme protocole de communication pour interagir avec les annuaires Active Directory. LDAP permet aux administrateurs de rechercher des objets Active Directory, de les créer, de les modifier ou de les supprimer. C'est un moyen standardisé et largement utilisé pour interagir avec les données stockées dans Active Directory.

Voici comment cela fonctionne généralement :

  • Les clients LDAP envoient des requêtes LDAP au serveur Active Directory.
  • Le serveur Active Directory répond aux requêtes en renvoyant les informations demandées, telles que des détails sur les utilisateurs, les groupes, les ordinateurs, etc.
  • LDAP peut également être utilisé pour effectuer des opérations de création, de mise à jour ou de suppression d'objets dans Active Directory, sous réserve des autorisations appropriées.

En résumé, LDAP est un protocole de communication utilisé pour accéder et gérer des annuaires, tandis qu'Active Directory est un service d'annuaire spécifique à Microsoft qui utilise LDAP comme protocole sous-jacent pour fournir des fonctionnalités de gestion d'annuaire dans les environnements Windows.

Qu’est-ce que l’authentification LDAP ?

LDAP v3 propose deux options pour l’authentification LDAP – simple et SASL (Simple Authentification and Security Layer).

L’authentification simple permet de mettre en œuvre trois mécanismes possibles d’authentification :

 

  • Authentification anonyme : accorde au client un statut anonyme pour accéder aux services LDAP.
  • Authentification non authentifiée : uniquement à des fins de journalisation, ne doit pas accorder d’accès au client.
  • Authentification par nom/mot de passe : accorde un accès au serveur en se basant sur les identifiants fournis – une simple authentification par nom/mot de passe n’est pas sécurisée et ne convient pas à une authentification dans les environnements sans protection de la confidentialité.

 

Il est important de noter que LDAP transmet par défaut tous ces messages en texte clair, ce qui fait que n’importe qui peut lire les paquets en utilisant un renifleur réseau. Vous devez ajouter un chiffrement de type TLS pour préserver la confidentialité de vos noms d’utilisateur et de vos mots de passe.
